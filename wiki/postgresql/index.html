객체-관계형 데이터베이스 관리 시스템.

제대로 사용한 적은 없고, Amazon Redshift를 통해서 쿼리 작성만 해보았다.

샌드박스 환경

도커로 Postgres를 실행해보자.

$ docker run -it --rm --name some-postgres -e POSTGRES_PASSWORD=password postgres

POSTGRES_PASSWORD는 필수 환경 변수이다. 컨테이너 내에서 접속할 거라 외울 필요는 없다.

이제 psql로 접속한다.

$ docker exec -ti some-postgres psql -U postgres

이제 기본적인 쿼리는 다른 SQL과 비슷하므로 사용하는데 큰 어려움은 없다.

MySQL 쿼리 차이점

MySQL과 다르게, group by로 aggregation 시 aggregation function을 사용하지 않는 컬럼을 select에 포함할 수 없다.

이를 해결하기 위해서, 모든 값을 보고싶다면 listagg를 사용하자. MySQL의 group_concat와 비슷하다.
아무 값이나 보고 싶다면 any_value를 사용하자. MySQL의 그냥 컬럼을 select하는 것과 비슷하다.

select
  family,
  listagg(name, ', ') as names,
  any_value(age) as age
from users
group by family;
